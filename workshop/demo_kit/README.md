# Operator demo kit

Presenter-facing materials for the 20-minute, operator-led, synthetic-only ClarionPI demo. See
[`../README.md`](../README.md) for the owned-synthetic rule.

## Contents
- [`disclosure.md`](disclosure.md) — **show/read first.** The demonstration disclosure (slide +
  speaker notes); the slide/verbal substitute for the product's S13 in-product labeling.
- [`talk_track.md`](talk_track.md) — the beat-by-beat 20-minute script (pre-run vs live marked).
- [`feedback_form.md`](feedback_form.md) — anonymous attendee feedback; collects no client/case info.
- [`paid_review_one_pager.md`](paid_review_one_pager.md) — the paid Arizona file-review offer
  (a separate written engagement).

## Presenter handoff checklist

Before the room:
- [ ] Fresh disposable database; owned-synthetic scenario only; no real data anywhere.
- [ ] `python workshop/scenarios/az_mva_01/generate.py` run; the eight PDFs ready to upload.
- [ ] Pre-run beats done; analysis, draft, and package exist to navigate.
- [ ] Screenshots / recording of the pre-run beats saved as a fallback.
- [ ] **Disclosure reviewed by a licensed attorney** and ready to show.

In the room:
- [ ] Disclosure shown/read before anything else.
- [ ] The operator drives; no attendee operates the product; no client information is entered.
- [ ] Deadline beat lands: private-party → 2-year AZ statute; public-entity deadline absent.
- [ ] Provenance click-through works for the demanded amounts.
- [ ] Feedback form + paid-review one-pager handed out.

## Scope note

This kit substitutes a spoken + slide disclosure for the shipping product's S13 in-product labeling.
That substitution is valid only for operator-led sessions on synthetic data, with a disposable
database and no live intake. It does not apply the moment anyone else drives the product, or any real
matter is involved.
